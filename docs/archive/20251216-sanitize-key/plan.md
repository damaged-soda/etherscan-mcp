# 去除敏感信息 技术变更计划（Plan）

## 0. 背景与目标（简短即可）
- 背景：SOT 文档中误写入真实 API Key，存在泄露风险。
- 目标：移除文档中的敏感值，改为占位符；确保无明文密钥存留。
- 非目标：不改代码、不改功能逻辑。

## 1. 影响范围（必须）
- 影响的 repo：无代码改动，仅文档（docs）
- 影响的模块/目录/文件：docs/sot/overview.md
- 外部可见变化：文档中的示例使用占位符而非真实 key。

## 2. 方案与改动点（必须）
- repo: docs
  - 改动点：将 SOT 中的真实 API Key 替换为占位符 `<your-api-key>`。
  - 新增/修改的数据结构：无。
  - 关键逻辑说明：仅文本替换，确保无敏感信息。

## 3. 自测与验收口径（必须，可执行）
- 检查：grep 文档不包含真实 key。
- 通过标准：SOT 示例使用占位符，未包含密钥明文。

## 4. SOT 更新清单（必须）
- docs/sot/overview.md：示例中的密钥改为占位符。
- 其他：无。

## 5. 完成后归档动作（固定）
1) 按第 4 节更新 SOT
2) 将整个目录从 docs/wip/YYYYMMDD-topic/ 移动到 docs/archive/YYYYMMDD-topic/
